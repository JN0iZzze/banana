# Миграция слайдов 20Feb → newgen

## Контекст проектов

- **Источник (старая презентация):** обычно `OtherAIPro/20Feb` (в чатах может называться Feb20). Корень статики: `20Feb/public/` — пути вида `/images/...`, `/comb.png` и т.д.
- **Цель (этот репозиторий):** `newgen/public/` — те же URL от корня сайта (`/images/...`), компоненты слайдов: `src/presentation/slides/`, дек: `src/presentation/decks/mainDeck.ts`.
- Компонент слайда принимает **`SlideRenderProps`**: `{ slide, index, totalSlides }` (см. `src/presentation/types.ts`).

## Алгоритм для агента

1. **Найти и прочитать исходный файл** по пути от пользователя (например `src/slides/content/SomeSlide.tsx`). Если путь неполный — поиск по репозиторию 20Feb по имени файла.
2. **Собрать список ассетов:** все `src=` у `<img>`, `url()` в стилях, импорты картинок. Нормализовать до путей от `public/` (без домена).
3. **Скопировать файлы** из `20Feb/public/...` в `newgen/public/...` с сохранением структуры папок (`mkdir -p` при необходимости). Команда `cp` из терминала — обязательна, не описывать только «положите файлы».
4. **Если файла нет в 20Feb** — явно сообщить пользователю; **не** подставлять заглушки и выдуманные пути (политика проекта: честная ошибка вместо fallback).
5. **Собрать React-слайд** в стиле newgen:
   - Корень: `SlideFrame` + `SlideBackdrop` (`variant`, `accent` как у ближайшего по смыслу демо: editorial → grid, signal → mesh, cinema → spotlight).
   - Контент: `SlideContent`, при необходимости `SlideHeader` (meta: строка счётчика в том же формате, что у соседних слайдов, например `Demo slide XX / YY`), `SlideGrid` / `SlideColumn`, `Box`, `SlideSection`, `Text` из `src/ui/slides`.
   - Анимации: `Reveal` с пресетами из `RevealPreset` (`soft`, `enter-left`, `enter-right`, …), не тянуть `AnimatedBlock` и прочие обёртки из 20Feb.
   - Картинки: пути в `src` оставить как в Vite public (`/images/...`). Без лишних декоративных рамок/фона, если пользователь или дизайн-система этого не требуют.
6. **Зарегистрировать слайд** в `mainDeck.ts`: уникальный `id`, человекочитаемый `title`, `theme` (`editorial` | `signal` | `cinema`), `component`, опционально `notes` и **`preloadAssets`** со списком путей от корня (`/images/...`), если слайд тяжёлый по картинкам.
7. **Существующий слайд:** если в деке уже есть заглушка с той же темой — заменить `component` и обновить заметки; иначе добавить новую запись.

## Согласованность дизайна

- Типографика только через **`Text`** (`variant`, при необходимости `size` для `h1`, `context` для блоков на акценте).
- Отступы и сетка — токены и компоненты из `layout.tsx` (`SlideGrid` 12 колонок, `gap` sm/md/lg как в соседних слайдах).
- Не смешивать старые классы вроде `text-retro-text` с новой темой без необходимости; переносить **смысл** раскладки, не копировать классы один в один.

## Проверка

- `npm run build` в корне newgen после изменений.
- Убедиться, что удалены неиспользуемые импорты и не осталось битых путей к ассетам.
